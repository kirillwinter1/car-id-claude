# review/ — архитектурные ревью и рефакторинги

Сюда кладём:
- Код-ревью (что плохо, что предлагается изменить).
- Планы рефакторингов (без пользовательских изменений).
- Архитектурные рекомендации.

Формат имени: `YYYY-MM-DD_SHORT_TOPIC.md` (например `2026-04-21_TELEGRAM_REFACTORING.md`).

Отличие от `features/`: фича = функциональность для пользователя, review = внутреннее изменение кода.

После реализации рефакторинга — документ остаётся здесь как история решения.
