# Спек 2.4 — Telegram-бот: deep-link онбординг

**Статус:** 📋 Design · **Заведён:** 2026-04-23 · **Эпик:** [2026-04-21_TELEGRAM_EPIC.md](2026-04-21_TELEGRAM_EPIC.md)
**Предусловие:** спек 2.3 замёржен в main.

## 1. Цель

Убрать шаг «поделиться контактом» из онбординга. В мобилке появляется кнопка «Настроить Telegram», которая открывает `https://t.me/<bot>?start=<signed_token>`. Бот распознаёт payload, достаёт `userId`, сразу привязывает `telegram_dialog_id` и показывает Home.

**Мотивация:** shareContact — плохой UX (три тапа с подтверждением системного диалога, в iOS показывается предупреждение «приложение запросило ваш номер»). Deep-link привязывает за один тап.

**Fallback:** старый shareContact-flow остаётся для пользователей, пришедших в бота напрямую (поисковый запрос в TG, рекомендация от знакомого).

## 2. Архитектура

```
mobile                                 backend                       telegram
  │                                       │                              │
  ├─ POST /api/telegram.get_start_url ───►│                              │
  │                                       │ generate signed token        │
  │◄─ {url: t.me/bot?start=<token>} ──────┤                              │
  │                                                                      │
  ├─ url_launcher.launch(url) ────────────────────────────────────────► /start <token>
                                          │                              │
                                          │◄── getUpdates ───────────────┤
  ◄─────────── "Добро пожаловать, Home" ──┤ TelegramRouter               │
                                          │  → parse /start payload       │
                                          │  → verify signature + TTL     │
                                          │  → bind telegram_dialog_id    │
                                          │  → render HomeScene           │
```

**Формат токена:** `base64url(userId:expiresAt).base64url(HMAC-SHA256(payload, secret)[0..10])`
- Пример: `NDI6MTc2MzI1MDAwMA.a1b2c3d4e5f6g7`
- Bounds: `userId` ≤ 19 символов (Long max), timestamp 10 цифр → payload ≤ 30 символов в base64url. Подпись 10 байт → 14 base64url. Итого ≤ 50 символов, в бюджете Telegram `/start` payload (64 символа).
- Секрет — `token.signing.key` (тот же HMAC secret, что и у JWT). Не вводим новую конфигурацию.
- TTL: 15 минут. Пользователь может не сразу нажать.

**Новые компоненты:**

- `TelegramStartTokenService` (пакет `auth/`) — `sign(userId, ttl)`, `verify(token) → Optional<Long userId>`.
- `TelegramAuthController` (пакет `controller/`) — `POST /api/telegram.get_start_url` возвращает `{url: "https://t.me/<bot>?start=<token>"}` для авторизованного пользователя.
- Изменение `TelegramRouter.extractAuthText` — если текст = `/start <payload>`, возвращает именно payload (без префикса).
- Изменение `TelegramAuthorizationService.handle` — новая ветка: если не телефон и не пустая, а похож на наш токен (содержит `.` и base64url символы) — пытается `verify` + bind. Если токен ок — привязывает и возвращает welcome. Если не ок — fallback на старый flow (просит контакт).

**Mobile (Flutter):**

- `NotificationsRepository.getTelegramStartUrl()` — POST `api/telegram.get_start_url` → `{url: ...}`.
- `notification_settings_screen.dart`: кнопка «Подключить бота» вместо `Utils.openUrl(TELEGRAM_BOT_URL)` дергает repo → открывает полученный URL. Если API 401/500 — fallback на `TELEGRAM_BOT_URL` (старый путь).
- После возврата пользователя на мобилку (`WidgetsBindingObserver.didChangeAppLifecycleState == resumed`) — перезагрузить `fetchNotificationSettings`. Если `telegramDialogId != null` — экран покажет «✅ Telegram подключён».

## 3. Формат токена — детали

```java
public record StartToken(long userId, long expiresAtEpochSec) {}

// sign:
// 1. payload = userId + ":" + expiresAt
// 2. hmac = HMAC-SHA256(payload, secret)
// 3. truncated = hmac[0..10]    // 10 bytes = 14 base64url chars
// 4. return base64url(payload) + "." + base64url(truncated)

// verify:
// 1. split on '.'  -> [b64payload, b64sig]
// 2. payload = decode(b64payload), parse userId:expiresAt
// 3. expected_sig = HMAC-SHA256(payload, secret)[0..10]
// 4. constant-time compare with actual sig
// 5. check expiresAt > now
// 6. return Optional.of(userId) or empty
```

**Telegram requirement:** start-payload — только `[A-Za-z0-9_-]`, не более 64 символов. Base64url использует `A-Za-z0-9-_` → совместимо. Точка `.` в payload НЕ разрешена Telegram'ом (проверено: только alphanumeric + underscore + hyphen). → **Используем `_` как разделитель**: `<b64payload>_<b64sig>`.

Итоговый формат: `NDI6MTc2MzI1MDAwMA_a1b2c3d4e5f6g7`.

**Безопасность:**
- 10-байтный truncated HMAC = 80 бит → brute-force практически невозможен (2^80 операций).
- TTL 15 минут ограничивает window attack.
- Constant-time compare защищает от timing-атак.
- Replay-атака: если кто-то перехватит URL и кликнет первым — привяжет ЕГО Telegram к нашему userId. Это атака уровня «кто-то видит экран мобилки + имеет TG». Не защищаемся отдельно: TTL 15 мин + короткий window ок.
- Если пользователь уже привязан (`telegramDialogId != null`) и кликает deep-link на другом TG-аккаунте → сервер выдаёт «уже привязан», не переписывает. Старый dialog_id защищён.

## 5. API

### `POST /api/telegram.get_start_url`

**Запрос:**
```json
{ "params": {} }
```
(авторизация по Bearer JWT)

**Ответ:**
```json
{
  "result": "true",
  "params": {
    "url": "https://t.me/car_id_test_bot?start=NDI6MTc2MzI1MDAwMA_a1b2c3d4e5f6g7"
  }
}
```

**Если `telegram.enable = false`** — 503 «Telegram не активирован».
**Если `telegram.bot` не настроен** — 500 server error.

## 6. Callback и pre-auth flow роутера

```java
// TelegramRouter.extractAuthText
if (message.getText() != null && message.getText().startsWith("/start ")) {
    String payload = message.getText().substring("/start ".length()).trim();
    return payload;
}
// обычный текст или контакт — без изменений

// TelegramAuthorizationService.handle
public SceneOutput handle(long chatId, String text) {
    // 1. если уже привязан — "уже привязан" (без изменений)
    if (settingRepository.existsByTelegramDialogId(chatId)) {
        return SceneOutput.send(messages.get("tg.auth.already_linked"), contactKeyboard());
    }
    // 2. попытка распарсить как start token
    Optional<Long> userIdOpt = startTokenService.verify(text);
    if (userIdOpt.isPresent()) {
        Long userId = userIdOpt.get();
        // check user exists
        Optional<User> user = userService.findById(userId);
        if (user.isPresent()) {
            settingRepository.updateTelegramDialogIdByUserId(userId, chatId);
            return SceneOutput.send(messages.get("tg.auth.welcome_deep_link"),
                ReplyKeyboardRemove.builder().removeKeyboard(true).build());
        }
        return SceneOutput.send(messages.get("tg.auth.token_user_not_found"), contactKeyboard());
    }
    // 3. классический phone flow — без изменений
    if (!MessageUtils.isValidPhone(text)) {
        return SceneOutput.send(messages.get("tg.auth.request_contact"), contactKeyboard());
    }
    // ... как было
}
```

**Ключи i18n:**
- `tg.auth.welcome_deep_link` — «✅ Готово. Ваш Telegram привязан к аккаунту Car-ID.»
- `tg.auth.token_user_not_found` — «Ссылка устарела или аккаунт не найден. Попробуйте снова из мобильного приложения.»

## 7. Безопасность чек-лист

- [x] HMAC-SHA256 с 256-битным ключом (тот же `token.signing.key` что и JWT).
- [x] TTL 15 мин — window атаки короткий.
- [x] 80 бит truncated HMAC — неподделываемо.
- [x] Constant-time compare — нет timing-атак.
- [x] `existsByTelegramDialogId` — не переписываем привязку, если chat уже привязан к ДРУГОМУ user.
- [x] `userService.findById` — не привязываем к несуществующему user.
- [x] Не логируем полный токен (лог может попасть в централизованный storage).

## 8. Что НЕ делаем

- **Отзыв токена** — TTL 15 мин достаточно, реестр не нужен.
- **Nonce / replay protection** — для 15-минутного window overkill; привязка идемпотентна (если chat уже привязан — игнорируем).
- **Уведомление пользователя** при повторной привязке с другого устройства — нет необходимости.
- **Multi-binding** — один user может иметь только один `telegram_dialog_id` (существующая model).
- **Нативная обработка deep-link в мобилке** — Flutter `url_launcher` просто открывает Telegram.
