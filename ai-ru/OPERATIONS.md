## Production сервисы

На prod-машине настроены 4 systemd-сервиса:

| Сервис | Что делает |
|--------|-----------|
| `car_id.service` | Запуск Spring Boot приложения |
| `deploy.service` | Заготовка под CI/CD (пока не используется) |
| `backup.service` | Ежедневный дамп БД в 03:00 по таймеру |
| `health.service` | Healthcheck каждые 5 секунд, при падении — уведомление в Telegram |

Конкретные пути, хост, пароли — в `ops.inf`.

## Деплой backend

Шаги (без конкретных путей):

1. Собрать jar: `gradle bootJar`.
2. Скопировать артефакт на prod-сервер через `scp`.
3. Перезапустить `deploy.service` через `systemctl restart`.

Полная последовательность с IP/путями — в `ops.inf`.

## Деплой frontend

1. Упаковать каталог `frontend/` в `tar.gz`.
2. Скопировать на сервер.
3. Распаковать поверх существующей директории.
4. Перезапустить nginx.

## SSL сертификаты

Обновляются раз в полгода/год. Основные шаги:

1. Получить у поставщика `certificate.key` и `certificate.crt` + цепочку `certificate_ca.crt`.
2. Собрать `certificate.pem` склейкой `certificate.crt` + двух блоков из `certificate_ca.crt`.
3. Скопировать `.pem` и `.key` в `/etc/nginx/certs` на сервере.
4. Перезапустить nginx.

Точное количество строк в блоках и команды — в `ops.inf`.

## Генерация QR-кодов (batch)

Генерация партии QR-стикеров — **ручная операция** через `main`-методы в утилитах backend. Запускается локально у разработчика.

### Этапы

1. **SVG-файлы.** Класс `ru.car.util.SvgUtils`, метод `main`:
   - задать имя директории для партии,
   - указать шаблоны из `BatchTemplates`, количество и номер batch для БД.
2. **Конвертация SVG → PDF.** Класс `ru.car.util.svg.PdfConverter`, метод `main`:
   - указать путь к директории с SVG.
   - результат: подпапки с PDF + склейка по подпапкам.
3. Полученные PDF передаются в типографию; номера QR уезжают в БД (батч).

**Замечания к коду** (техдолг):
- В коде захардкожен абсолютный путь разработчика — нужно актуализировать под свою машину перед запуском.
- Пара `main`-методов — не рекомендованный способ запуска, но текущая реализация.

Пошаговая инструкция с примерами — в `ops.inf`.
