#  features/ — реализованные фичи

Каждый файл — одна фича, формат `FN_SHORT_NAME.md` (например `F8_TELEGRAM_BOT.md`).

Нумерация сквозная, см. индекс в [../FEATURES.md](../FEATURES.md).

## Шаблон карточки

```markdown
# FN: Название
**Статус:** ✅ В проде · **Последний апдейт:** YYYY-MM-DD

## Что делает
1-2 предложения для тех, кто видит фичу впервые.

## Сценарии
1. Шаг — 2. Шаг — 3. Шаг

## API / Интеграция
- `METHOD /path` — короткое описание
- Внешний сервис: имя, назначение

## Реализация
- **Backend:** `package.Class`, `package.Class`
- **Mobile:** `lib/screens/...`
- **Frontend:** `*.html` (если есть)
- **БД:** таблицы / миграции (если релевантно)

## Ограничения / известный техдолг
- Что работает не идеально, что планируется рефакторить

## Ссылки
- Связанные фичи: FN, FN
- Обсуждение/ревью: `review/...` (если есть)
```

## Правила

- Карточка — **не заменяет** код или его чтение. Это шпаргалка на 30-100 строк.
- Пиши для того, кто знает проект на уровне README, но не знает эту фичу.
- Если фича эволюционировала — отметь в разделе "Ограничения" или перенеси старую версию в `archive/`.
- Дата в заголовке = последнее содержательное обновление документа, а не кода.
