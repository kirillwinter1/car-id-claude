# BF3: Редизайн лендинга car-id.ru

**Статус:** 📋 Planned (Phase 4 в [ROADMAP.md](../ROADMAP.md))
**Последний апдейт:** 2026-04-21

## Что делаем

Переделываем `frontend/index.html` (+ публичные страницы: offer, privacy-policy) в современный, конверсионный лендинг. Цель — повышение вероятности «прохожий увидел стикер → QR → car-id.ru → купил на Ozon/WB».

## Мотивация

- Текущий лендинг — минимальный. Нет видеодемо, нет отзывов, нет сценариев использования.
- Основной продукт продаётся на Ozon (см. [ozon-workspace](../../../ozon-workspace/), reference memory). Отзывы и продажи оттуда можно подтягивать в лендинг.
- После Phase 3 (мульти-канальная auth) у нас появится больше способов «затянуть» посетителя — показать ссылку на Telegram-бот, MAX, etc.

## Желаемое наполнение

1. **Hero-блок.** Одно сообщение «QR-стикер на машине — получайте уведомления от соседей и очевидцев». Ссылки на Ozon / RuStore / App Store.
2. **Сценарии использования** (карточками):
   - ЖК: «сосед мешает» → push.
   - Сервис: мойка/автосервис уведомляет о готовности.
   - ДТП: очевидец быстро связывается.
   - Эвакуатор / ГИБДД.
3. **Видеодемо** — 20-30 сек как работает: сканирование → отправка → получение push.
4. **Отзывы** — подтягиваются через MCP-сервер Ozon (из `ozon-workspace/analytics/reports/`), обновляются автоматически.
5. **FAQ** — что такое QR-стикер, зачем приложение, безопасно ли это, сколько стоит.
6. **B2B-блок** — для автомоек, сервисов, каршеринга (подсказывает возможный интеграционный сценарий).
7. **Призывы к действию** — кнопки Ozon/WB + QR-коды на маркеты для телефона.
8. **Подвал** — privacy, offer, контакты, ссылка на Telegram-бот.

## Технические решения

- Остаться на vanilla JS (как сейчас) или перейти на что-то лёгкое (Astro / Eleventy) — решается во время дизайн-фазы.
- Минификация CSS/JS — добавить build-шаг (см. [TECH_DEBT W2](../TECH_DEBT.md)).
- CSP headers — добавить в nginx (см. [TECH_DEBT S11](../TECH_DEBT.md)).
- WebSocket вместо polling статуса уведомления — см. [TECH_DEBT W1](../TECH_DEBT.md).

## Предусловия

- Phase 3 завершена (мульти-канальная auth — чтобы CTA-блок на лендинге показывал все способы входа).
- MCP-сервер Ozon работает стабильно (для подтягивания отзывов).
